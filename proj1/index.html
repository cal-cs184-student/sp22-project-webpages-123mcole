Task 2

The supersampling algorithm is fed a point in the triangle, then goes through two for loops that go through each super sample point in that point, and computes the “innernes” of the point in the triangle using linear interpolation. Then the results of whether a supersample point was in the triangle (binary) are averaged for a weight used in computing the final color from the original color.

The reason why, as the sampling rate goes up, the smoothness of the edges go up, is because we are decreasing the size of the “computational pixel” and averaging over the results. Thus, we get less of a sudden change in value over pixels and a more smooth transition aided by the averaging of the precise values that campure the gradual change in color.

Task 5

Pixel sampling includes computing the barycentric coordinates, then to uv coordinates. This allows us to sample points from a texture because the texture is made up of equilateral triangles. Bilinear interpolation sampling involves taking the closest 4 surrounding pixels, and averaging the color of each pixel based on how close it is to each pixel; this is a weighted sum based on distance. Nearest interpolation sampling simply involves grabbing the nearest pixel.
